# 安全与隐私（desktop-pet）

> **初稿**：实现前威胁建模；随功能合并持续更新。

## 1. 数据最小化（原则）

- 默认不向开发者服务器上传用户屏幕或第三方窗口内容。  
- 若实现「前台窗口标题」等能力：**默认关闭**，开启时显著告知，并写入隐私说明。

## 2. 密钥与凭证

- LLM API Key、OAuth 等：**禁止**硬编码；使用 OS 凭据存储或加密本地存储（实现后写明具体 API）。  
- Agent **不得**在仓库中提交真实密钥；仅使用占位符与 `.env.example`。

## 3. 插件与脚本

- 插件须在 **白名单能力** 与 **沙箱/子进程** 下运行；禁止任意文件系统与任意子进程（除非单独高危开关 + 确认）。  
- 对外网络请求须可审计（域名白名单或用户确认）。

## 4. 更新与供应链

- 依赖锁定（lockfile）；安全 PR 走 [agent-playbook.md](agent-playbook.md) 双子流程。  
- 发布产物记录哈希；可选代码签名由 **Human** 执行。
