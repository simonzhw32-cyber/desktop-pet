# Skin Pack 创作说明（`rich-v1`）

应用引擎只认 **逻辑状态 + 帧序列**；像素资源全部由 Skin Pack 提供。本项目 **`coreProfile` 冻结为 `rich-v1`**。

## 1. 强制状态键（缺一不可）

下列键必须在 `skin.json` 的 `states` 中出现，且帧文件齐全、`canvas` 一致：

`idle`，`walk`，`click`，`drag`，`sleep`，`wake`，`hover`，`happy`，`sad`，`surprised`，`wave`

任一缺失 → 整皮校验失败（回退内置默认皮）。

## 2. 扩展状态（可选）

可在 `states` 中增加任意额外键（如 `eat`、`dance`）。未实现某扩展键时，触发该动作会 **回退**（默认 `idle`，应用内可配置）。

## 3. 目录结构

```text
your-skin/
  skin.json
  assets/
    idle_001.png
    ...
```

## 4. `skin.json` 建议字段

| 字段 | 说明 |
|------|------|
| `schemaVersion` | 皮肤清单格式版本 |
| `coreProfile` | 必须为 `rich-v1`（与本引擎一致） |
| `id` / `name` / `author` | 元数据 |
| `preview` | 可选，缩略图路径 |
| `canvas` | `width`、`height`，**所有帧同一尺寸** |
| `anchor` | 锚点（建议脚底中心），避免角色「飘」 |
| `states.<key>.frames` | 有序帧文件名列表 |
| `states.<key>.fps` 或 `durationMs` | 时序 |
| `states.<key>.loop` / `oneshot` | 循环或单次回到 idle |
| `hit` | 可选，命中近似形状 |

## 5. 导出建议

- 格式：**PNG**，透明背景（RGBA）。  
- 画布：例如 **512×512** 或 **768×768**，全套统一。  
- 脚底对齐同一水平线，便于锚点。

## 6. AI 作图提示词（外部工具使用）

### 6.1 角色一致性基准

> 一只【物种/性格关键词】的 Q 版桌面 mascots，全身可见，站立姿势，粗描边或无边线扁平二次元任选其一，【配色】，正面略四分之三视角，适合循环小动画，**纯白背景**，高分辨率，角色居中，**从头到脚完整**，不要裁切脚部。

### 6.2 负向提示（通用）

> 文字，水印，对话框，复杂背景，地面阴影（若你希望阴影自建另议），模糊，低分辨率，多余肢体，解剖错误。

### 6.3 `rich-v1` 分状态要点（锁定同一角色参考后再出序列帧）

| 状态键 | 中文提示词要点 |
|--------|----------------|
| idle | 【角色】轻微呼吸感待机，手臂自然下垂，表情平静，**4～8 帧循环**，全身同一站位，姿态变化极小 |
| walk | 【角色】原地踏步或小幅度左右摆动，**循环**，足部交代清楚，头顶高度基本不变 |
| click | 【角色】被戳一下：惊讶或歪头，**一次性**，末帧接近 idle |
| drag | 【角色】被抓起悬空：脚离地、略蜷缩或慌张，**可循环小幅挣扎**，悬空高度稳定 |
| sleep | 【角色】闭眼打盹；不要 UI 元素则「Zzz」放入负向提示 |
| wake | 【角色】被叫醒：揉眼或猛然抬头，**一次性**，结束接近 idle |
| hover | 【角色】鼠标靠近：轻探头或耳朵竖起，**短循环或小幅度** |
| happy | 【角色】开心：笑眼、小幅跳或拍手，**一次性** |
| sad | 【角色】委屈：耳朵下垂或湿润感（无文字），**一次性** |
| surprised | 【角色】惊讶瞪眼、后退半步，**一次性** |
| wave | 【角色】单侧挥手打招呼，**一次性** |

### 6.4 扩展动作通用句式

> 【角色】+ 【具体动作】+ 全身完整 + 统一画布 + **loop/oneshot 与帧数** + 纯白背景；与 idle 同画风、同头身比。

### 6.5 合成与命名

> 多帧放入同一正方形画布，底部对齐；导出 `idle_001.png`…；在 `skin.json` 填写 `fps` / `oneshot`，并声明 `"coreProfile": "rich-v1"`。
